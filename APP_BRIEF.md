# 🏭 APP_BRIEF: Vertex AI RAG Engine Tutorial

> **Status:** 🔒 LOCKED  
> **Version:** 1.0 (Full Lifecycle)  
> **Project:** Managed RAG Migration (Gemini File Search -> Vertex RAG Engine)

---

## 1. Mission Statement
Create a 4-lesson progressive Python tutorial demonstrating the **Vertex AI RAG Engine**. This project replicates the pedagogical structure of the previous "File Search API" tutorial, allowing a developer to move from a clean environment to a grounded AI response using enterprise-grade managed RAG.

## 2. Hero Action
The user executes four sequential CLI scripts (lesson_1 through lesson_4) to initialize the environment, provision a vector corpus, ingest local documents, and generate a grounded answer—all while maintaining state through a local `corpus_name.txt` file.

## 3. User & Auth Scope
* **Persona:** AI Engineer / Developer.
* **Auth:** **Application Default Credentials (ADC)** only via `gcloud auth application-default login`.
* **Environment:** WSL/Linux/macOS with `google-cloud-aiplatform` installed.
* **Strict Requirement:** Project configuration (ID, Location) must be loaded via `.env`.

## 4. The Lesson Pipeline (Functional Requirements)

### Lesson 1: Initialization & Auth
* **Goal:** Verify the environment.
* **Logic:** Load `.env`, initialize `vertexai`, and check ADC.
* **Output:** Success message and environment summary.

### Lesson 2: Corpus Provisioning
* **Goal:** Create the managed vector database.
* **Logic:** Instantiate a `RagCorpus` using `text-embedding-004`.
* **Persistence:** Write the full resource name to `corpus_name.txt`.

### Lesson 3: Document Ingestion
* **Goal:** Index local knowledge.
* **Logic:** Read `corpus_name.txt`, scan `/uploads`, and call `rag.import_files`.
* **Chunking:** Must use `chunk_size=512` and `chunk_overlap=100`.

### Lesson 4: Grounded Generation (RAG)
* **Goal:** Prove the RAG system works.
* **Logic:** Read `corpus_name.txt`, create a `RagRetrievalTool`, and call `generate_content`.
* **Model:** Use `gemini-2.0-flash-001`.

## 5. Scope Locks (The Guardrails)

| ✅ IN SCOPE | 🚫 OUT OF SCOPE |
| :--- | :--- |
| Independent .py scripts | No Web UI (FastAPI, Streamlit) |
| .env for configuration | No external Vector DBs |
| Eyesight-Aware formatting | No interactive chat loops |
| Local file imports from /uploads | No API Key auth (GEMINI_API_KEY) |
| corpus_name.txt state handoff | No deployment pipelines |

## 6. Success Definition
1. **Zero Guessing:** Codex builds all 4 lessons without asking for IDs.
2. **State Integrity:** lesson_4.py runs using the file generated by lesson_2.py.
3. **Readability:** Explanations ALWAYS come BEFORE code blocks.
